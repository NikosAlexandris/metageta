 16  To add support for another format: 
 17  ================================== 
 18  -       Create a new .py file. Name the .py file whatever you want as long as the filename doesn't begin with an underscore "_" 
 19  -       Define one or more "regular expression" strings to match your formats filename 
 20  -       Create a "Dataset" class that inherits either from the base __dataset__.Dataset class or from the __default__.Dataset class (See class hierarchy below) 
 21  -       The default Dataset class is useful if GDAL can read your format and you just need to populate some extra fields.  
 22  -       Populate appropriate metadata and extent variables in the __init__ method of your dataset class 
 23  -       Your format will be automatically loaded when the formats module is initialised. 
 24  -       Errors should be propagated back up the chain. If you can't handle a certain file and for some reason you don't want an error to get raised (eg. the ENVI driver (*.hdr) doesn't handle ESRI bil/flt headers (*.hdr)) then raise NotImplementedError which will be ignored in lib.formats.Open() 
 25  -       If you want some info to get logged by the application and then continue processing (e.g the image doesn't have a projection defined, etc...) then use the warnings.warn("Some message") method - don't forget to import the warnings module! 
 26  -       Date/Time formats must follow follow AS ISO 8601-2007 (see: U{http://www.anzlic.org.au/metadata/guidelines/date_and_datetime.htm})
