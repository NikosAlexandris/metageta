 23  ==============

 26  Adding another stylesheet 
 27  ------------------------- 
 28      - Create a new/add an existing .xsl file that transforms crawler output (schema described below) to your XML schema.  
 29      - Create an xml file that describes your XML schema (see below) 
 30      - Save the xsl & xml file to the lib/transforms directory, it will be automatically available 
 31  Note: You may also pass the filepath to an external XSL stylesheet to the L{Transform} function. 
 32   
 33  Metadata schema 
 34  --------------- 
 35  The following schema defines a simple XML format which holds extracted metadata  
 36  (see L{DictToXML} and L{ListToXML}) to be transformed by an XSL stylesheet to  
 37  various XML metadata schemas:: 
 38      <?xml version="1.0" encoding="UTF-8" ?> 
 39      <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> 
 40        <xs:element name="crawlresult"> 
 41          <xs:complexType> 
 42            <xs:sequence> 
 43              <xs:element ref="{FIELD}" /> 
 44            </xs:sequence> 
 45          </xs:complexType> 
 46        </xs:element> 
 47        <xs:element name="{FIELD}"> 
 48          <xs:complexType mixed="true" /> 
 49        </xs:element> 
 50      </xs:schema> 
 51  Where: {FIELD} are the fields defined in L{formats.fields}, plus any additional metadata elements you pass in.  
 52  For example, you can pass additional metadata elements in by adding them to the spreadsheet  
 53  which is passed to L{runtransform.py<runtransform>}.   
 54  Additional metadata elements for existing XSL stylesheets will be documented below. 
 55   
 56  Stylesheet description schema 
 57  ----------------------------- 
 58  The following schema defines an XML file that describes an XSL stylesheet to be used by this module 
 59  to transform from the simple XML format defined above to various XML metadata schemas, 
 60  e.g. the ANZLIC Metadata Profile:: 
 61      <?xml version="1.0" encoding="UTF-8" ?> 
 62      <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> 
 63        <xs:element name="stylesheet"> 
 64          <xs:complexType> 
 65            <xs:attribute name="name" type="xs:string" use="required" /> 
 66            <xs:attribute name="file" type="xs:string" use="required" /> 
 67            <xs:attribute name="description" type="xs:string" use="required" /> 
 68          </xs:complexType> 
 69        </xs:element> 
 70      </xs:schema>
