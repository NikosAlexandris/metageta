Module Hierarchy

  • crawler: Iterator for metadata crawling.
  • formats: Generic and custom image format drivers
    • formats.__dataset__: Defines the metadata fields and populates some basic info
    • formats.__default__: Metadata driver for generic imagery formats including GDAL Virtual Rasters (VRT files and xml strings)
    • formats.__fields__: Dictionary of field names and data types
    • formats.ali_hyperion: Metadata driver for EO1 ALI (L1G & L1R) & Hyperion (L1R) images
    • formats.alos: Metadata driver for ACRES ALOS AVNIR-2/PRISM/PALSAR imagery
    • formats.aster: Metadata driver for ASTER imagery
    • formats.ccrs: Metadata driver for ACRES Landsat CCRS/SPOT 1-4 imagery
    • formats.ecw: Metadata driver for ECW imagery
    • formats.envi: Metadata driver for ENVI imagery
    • formats.esribil: Metadata driver for ESRI Bil imagery
    • formats.esrigrid: Metadata driver for ESRI GRIDs
    • formats.fast_l7a: Metadata driver for ACRES Landsat FastL7A imagery
    • formats.jp2: Metadata driver for JPEG2000 imagery
    • formats.netcdf: Metadata driver for NetCDF imagery
    • formats.nitf: Metadata driver for NITF imagery
    • formats.quickbird: Metadata driver for Digital Globe Quickbird imagery
    • formats.spot_cap: Metadata driver for SPOT 1-4 imagery
    • formats.spot_dimap: Metadata driver for SPOT DIMAP imagery
  • geometry: Geometry and dataset helper functions
  • overviews: Generate overviews for imagery
  • progresslogger: Provide GUI & file progress logging
  • runcrawler: Script to run the Metadata Crawler
  • runtransform: Contains code to show GUI to gather input arguments when none are provided To run, call the eponymous batch file which sets the required environment variables
  • spatialreferences: Spatial reference helper functions
  • splashscreen: No, they don't take long at all to load when run from C:\ drive.
  • transforms: Utility functions to assist XSL transforms
  • utilities: Utility helper functions
